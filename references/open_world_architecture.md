# 开放世界模式 — 架构分析 v2.0

## 核心理念

开放世界模式的终极目标：**像一个真实世界运行**。

- 角色在其中自主行动、交互、碰撞
- 世界模型保证基本规则不被打破
- 因果律（天意）作为大手，推动主线、确保关键事件发生
- 但角色在锚点约束范围内拥有真正的自由——可能做出大纲从未预料的选择

---

## 一、Skill = 引擎，Project = 世界实例

这是理解整个系统最根本的分界线。

### Skill（引擎）— 固定不变

```
story-studio/
├── SKILL.md                         ← 系统说明书
├── references/agent_prompts/         ← Agent角色定义（纯模板，无故事内容）
│   ├── 统筹.md                       ← 每场戏前决定剧组构成
│   ├── 角色导演.md                   ← 分析出场需求、激活NPC
│   ├── 因果律.md                     ← 主线推进、天意干预
│   ├── 世界模型.md                   ← 只读世界规则、事实核查
│   ├── 现场监理.md                   ← 入场许可关卡
│   ├── 总导演.md                     ← Scene Spec、锚点、回合管理
│   ├── 角色Agent模板.md              ← 角色扮演通用模板
│   ├── 审查官.md                     ← OOC/AI味/冲突检测
│   ├── 摄影.md                       ← 原始回合记录
│   ├── 剪辑.md                       ← 素材编排
│   └── 特效.md                       ← 文学润色
├── references/
│   ├── workflow_patterns.md          ← 工作流模式库（A/B/C/D/E）
│   ├── review_criteria.md            ← 评审标准
│   └── anchor_system.md              ← 锚点系统说明
└── templates/                        ← 项目初始化模板
    ├── PROJECT_MAP.md
    ├── character_folder/             ← 角色文件夹骨架
    └── scene_spec.yaml               ← Scene Spec格式
```

**Skill 包含什么**：
- Agent的**角色定义**（"审查官做什么"——不是"审查官审查周远"）
- **流程和规则**（"如何检查OOC"——不是"周远不能跑"）
- **模板和格式**（"角色档案应包含7个维度"——不是"周远的档案"）
- **工作流模式**（"双人对话模式由这些Agent组成"——不是"周远和陈念的对话"）

**Skill 不包含什么**：
- 任何具体故事的设定、角色、情节
- 任何具体世界的规则（那不是引擎的事，是项目的事）

### Project（世界实例）— 每个故事一个

```
workspace/{书名}/
├── world_bible.md                   ← **这个世界的规则**（世界模型读取此文件）
├── volumes/                         ← 卷蓝图（因果律读取此文件）
│   └── volume1_blueprint.md
├── characters/                      ← **这个世界的居民**（角色导演读取此目录）
│   ├── INDEX.md
│   └── {角色名}/
│       ├── profile.md               ← 固定档案（角色Agent的"人格"）
│       ├── agent.md                 ← Agent Prompt（基于profile+模板生成）
│       ├── state.md                 ← 实时状态（每场戏后更新）
│       ├── timeline.md              ← 出场时间表（现场监理强制执行）
│       └── memories.md              ← 角色记忆（知识边界检查）
├── specs/                           ← Scene Spec（总导演生成+读取）
├── scenes/                          ← 原始素材（摄影写入+剪辑读取）
├── chapters/                        ← 成品章节
├── reviews/                         ← 评审报告
├── tracker/                         ← 进度+伏笔+冲突
└── PROJECT_MAP.md                   ← 自描述清单
```

**Project 包含什么**：
- 这个世界的**物理/社会/科技规则**（world_bible.md）
- 这个世界的**居民**（characters/）
- 这个故事的**命运蓝图**（volumes/）
- 运行时的**状态和产物**（scenes, chapters, tracker）

## 二、Agent运行时如何读取项目

每个Agent启动时，**读取项目中的对应文件**，而不是在Skill中硬编码故事内容。

### 示例：世界模型的运行

```
Skill中的世界模型Agent Prompt（模板）：
  "你是世界模型。你只能回答 world_bible.md 中有记录的事实。未知即拒绝。"
  
Agent启动时：
  1. 读取 {project_path}/world_bible.md
  2. 现在它知道了"这个世界的规则"
  3. 被问："这个地点有什么特征？" → 查 world_bible → 回答或拒绝
  4. 被问："这个角色能飞吗？" → 查 world_bible 的物理规则 → 回答或拒绝
```

### 示例：角色导演的运行

```
Skill中的角色导演Agent Prompt（模板）：
  "你是角色导演。分析场景需求，决定哪些角色出场、激活哪些NPC。"
  
Agent启动时：
  1. 读取 {project_path}/volumes/volume{N}_blueprint.md（当前幕的需求）
  2. 遍历 {project_path}/characters/*/timeline.md（每个角色的出场安排）
  3. 分析："本场景需要{角色A}（已授权）+ {NPC类型}（需要激活）"
  4. 返回角色清单给统筹
```

### 示例：现场监理的运行

```
Skill中的现场监理Agent Prompt（模板）：
  "你是现场监理。逐项检查：出场许可/世界设定/状态一致/锚点完整。"
  
Agent启动时：
  1. 读取当前场景的 Scene Spec
  2. 读取 world_bible.md（核查场景细节是否有依据）
  3. 读取 characters/{角色}/timeline.md（核查出场授权）
  4. 读取 characters/{角色}/state.md（核查状态连续性）
  5. 输出：通过/拒绝 + 具体原因
```

## 三、开放世界模式的完整执行流程

### 场景开始前（统筹在主会话中执行）

```
1. 统筹读取 volume{N}_blueprint.md
   → 确定当前场景的目标和上下文

2. 统筹 → 角色导演：
   "本场景概要：{从blueprint提取}。分析出场需求。"
   
   角色导演执行：
   a. 遍历 characters/*/timeline.md → 确认谁授权在场
   b. 分析场景功能需求 → 确认需要激活的NPC/AI助手
   c. 分级决策：重要角色→独立Agent / NPC→杂鱼Agent
   d. 返回：{角色列表, NPC列表, 备注}

3. 统筹 → 因果律：
   "本场景蓝图节点：{must_happen}。预设天意干预触发条件？"
   
   因果律执行：
   a. 读取 blueprint 中本场景的关键事件
   b. 标记"如果角色未在{n}回合内达成{must_happen}，触发干预"
   c. 预设干预方式（信息揭示/外部威胁/机遇/身体变化）
   d. 返回：{干预计划}

4. 统筹 → 世界模型：
   "本场景涉及：{地点/时间/道具/角色能力}。请提供依据。"
   
   世界模型执行：
   a. 读取 world_bible.md
   b. 逐项核查并返回出处
   c. 标注"未记录"的项目（需补充设定或拒绝）
   d. 返回：{事实核查报告}

5. 统筹选择工作流模式：
   - 单人场景 → 模式A
   - 包含对话/冲突 → 模式B
   - 多角色+复杂环境 → 模式C
   - 动作/逃亡 → 模式D
   - 支线独立故事 → 模式E

6. 统筹组建Team并生成Agent：
   TeamCreate("{项目名}-{场景id}")
   
   根据模式生成Agent：
   - 角色Agent（从 project/characters/{角色}/agent.md 加载）
   - 总导演（从 skill/references/agent_prompts/总导演.md 加载）
   - 审查官、摄影、因果律、世界模型、现场监理等按需加载
```

### 场景执行中（Team内Agent通信）

```
[总导演] 基于统筹决策生成 Scene Spec + 锚点约束
[现场监理] 入场许可检查 → 通过/拒绝
[总导演] 注入锚点 → 宣布场景开始
[角色Agent A] 回合1：观察→思考→行动→对话→目标
[审查官] 回合1 OOC检查 → 报告总导演
[总导演 → 角色Agent B] 回合2开始
[角色Agent B] 回合2：...
[摄影] 持续记录所有回合
[因果律] 监测：角色是否偏离蓝图？需天意干预？
[总导演] must_happen达成？→ 宣布场景结束
        未达成？→ 循环或通知因果律触发干预
```

### 场景结束后（后期管线）

```
[总导演] 更新所有角色 state.md + memories.md
[摄影 → 剪辑] 传递原始回合素材
[剪辑] 编排素材 → 删减/合并/过渡 → 生成章节草稿
[特效] 文学润色 → 动词升级/感官补充/AI味净化
[审查官] 全面审查 → P0/P1/P2 + 冲突检测 + 伏笔追踪
[5角色评审] 量化评分 → 通过/修订
```

## 四、关键原则

### 原则1：Skill永不含故事

审查官的Prompt中可以写"检查角色行为是否OOC"，但不能写"检查周远是否跑了"。
世界模型的Prompt中可以写"只能回答world_bible中的事实"，但不能写"德令哈在青海"。

所有示例用 `{占位符}`：`{角色名}`、`{地点}`、`{道具}`、`{时间}`。

### 原则2：Agent启动时读取项目

Agent不是"预装"了故事知识的。它的知识只在项目文件中。
Skill定义了"如何读取"，Project提供了"读什么"。

### 原则3：世界规则在项目中，不在Skill中

world_bible.md 是这个世界的宪法。世界模型是宪法的执行者，不是宪法的制定者。
如果world_bible中没有某个规则——团队讨论、创建、写入world_bible，然后世界模型才能执行它。

### 原则4：开放世界 = 约束下的自由

角色Agent拥有：在锚点约束范围内，完全自主决策。
角色Agent没有：突破锚点、知道剧情走向、看到其他角色内心、知道天意干预。

世界模型的规则 + 锚点约束 = 物理引擎
因果律的天意干预 = 叙事引力
角色Agent的自主决策 = 自由粒子

三者在场景中同时运行，产生的结果可能是任何人——包括用户和统筹——都没有预料到的。这是开放世界模式真正的价值。
