# Story Studio v2.0 快速启动指南

## 两种执行模式

| 模式 | 触发词 | 说明 |
|------|--------|------|
| **串行模拟**（默认） | "启动剧组" / "开拍" | 主AI依次扮演所有Agent，适合快速原型和单章测试 |
| **Team并行**（完整） | "启动剧组 Team模式" | 每个Agent独立进程运行，角色真实博弈，适合长篇连载 |

---

## 启动方式

### 串行模拟（快速）

**新项目**：
- "启动剧组"
- "开始写小说"
- "开拍"

**恢复已有项目**：
- "恢复《书名》"
- "继续写《书名》"
- 直接说项目名，Skill 会读取 PROJECT_MAP.md 自动恢复状态

### Team并行（完整）

**启动**：
- "启动剧组 Team模式"
- "Team模式写《书名》"

系统会：
1. `TeamCreate("书名-剧组")`
2. 逐个启动12个Agent（每个加载独立prompt+上下文）
3. 因果律主持创世会议
4. 进入Agent间SendMessage通信循环

> ⚠️ Team模式消耗更大的上下文和计算资源。建议：短篇/测试用串行模拟，长篇连载用Team并行。
> 详细编排流程见 `references/team_orchestration.md`

## 五阶段工作流

### Phase 1: 创世

```
你：启动剧组，我想写一个新故事

因果律：创世会议开始——
1. 类型：玄幻/科幻/都市/历史/悬疑/其他？
2. 核心命题：你想让读者思考什么问题？
3. 规模：短篇(1卷) / 中篇(3卷) / 长篇(5卷+)
4. 基调：黑暗/温暖/悬疑/热血/黑色幽默？
5. 一句话卖点？

你：科幻，探讨"数字永生与死亡权利"，3卷中篇，偏黑暗文艺

世界模型：开始建立世界圣经——
需要确认：
1. 时间设定（近未来/远未来/架空）
2. 核心科技/力量体系
3. 社会制度
4. 主要地理舞台

（逐一回答后生成 world_bible.md）
```

### Phase 2: 卷前筹备

```
因果律：第1卷即将开始。回顾计划——
卷一核心命题：XXXXX

请确认：
1. 本卷核心事件：必须发生的大事？
2. 感情线：角色关系如何发展？
3. 终点：本卷结束时的状态？
4. 需要引入几个新角色？

（回答后生成 volume1_blueprint.md + 各角色 timeline.md）

角色导演：分析需求...
需要以下角色类型：
- 男主：濒死的科学家
- 女主：年轻的逃学生
需要生成候选吗？还是你有明确想法？

（选角完成 → 生成 characters/{角色名}/ 完整文件夹）

总导演：拆解蓝图...
已生成 18 个 Scene Specs，因果律审核中。

因果律：场景概览如下——
第一幕：{3场戏概要}
第二幕：{4场戏概要}
...
确认后开始第一场戏。
```

### Phase 3: 场景执行

```
总导演：提交场景启动申请 [scene_001_spec.yaml]

现场监理：执行入场许可检查——
- 出场许可：✅ {角色} 第1幕授权
- 世界设定：✅ 在 world_bible §X 有记录
- 状态一致：✅
- 锚点完整：✅
→ 入场许可通过。签发。

总导演：宣布场景开始。
  ## 场景 scene_001：{场景名}
  ### 锚点约束
  - 时间：XXXX
  - 地点：XXXX
  - 知识边界：[角色知道/不知道什么]
  - 物理限制：[角色当前能力]
  - 禁止：[绝对不能做的事]
  ### 初始情境
  {详细场景描述}
  ### 回合开始。

{角色Agent}：回合 1 — [观察→思考→行动→对话]

审查官：回合间隙检查 ✅ 无OOC/无AI味

{角色Agent}：回合 2 — [...]

[用户随时可介入]
你：制造冲突
因果律：触发天意干预——门铃响了。

{角色Agent}：回合 3 — 回应门外情况

总导演：must_happen 全部达成。场景结束。
  更新角色状态...
  {角色} state.md 已更新

审查官：场景结束审查——
P0: 0 | P1: 1（抽象心理描写）| 通过
```

### Phase 4: 章节生成

```
（几场戏完成后）

你：生成第一章

剪辑：整理素材...
从 scene_001-004 中筛选，保留 85% 内容
删减 {n} 回合冗余对话
添加过渡

特效：文学润色...
动词升级 {n} 处
感官补充 {n} 处
AI味清理 {n} 处

审查官：全面审查——
设定一致性：✅
时间线正确：✅
伏笔追踪：2个新埋，待回收
冲突检测：无冲突

5角色评审团：
阅读者：88/100 ✓
编审：91/100 ✓
故事家：85/100 ✓
文学顾问：89/100 ✓
毒舌读者：87/100 ✓
→ 综合：85/100 — 优秀，可发布

你：审阅 → 确认 / 要求修改
```

### Phase 5: 卷末复盘

```
你：卷末复盘

审查官：卷1一致性报告——
设定矛盾：0
角色OOC：2次（均已修正）
伏笔状态：埋入8个，回收4个，待回收4个
常见问题：抽象心理描写偏多（建议下卷关注）

因果律：卷1完成。回顾终点 vs 蓝图终点——
✅ 一致。下卷调整方向...

世界模型：world_bible v1.0 → v1.1
新增条目：{卷1中新出现的设定}

角色记忆蒸馏：
{角色A} 37条记忆 → 5条核心记忆
{角色B} 28条记忆 → 4条核心记忆

状态检查点已存档 → 可从卷2直接恢复。
```

## 关键规则（务必遵守）

1. **设定文件必须先存在**：没有 world_bible.md 不要开始写
2. **现场监理不可绕过**：每场戏必须通过入场许可
3. **锚点不可违背**：角色Agent必须在锚点范围内行动
4. **审查不可省略**：P0问题 = 流水线暂停
5. **状态必须更新**：每场戏后更新所有相关角色的 state.md

## Team并行模式工作流速查

```
你："启动剧组 Team模式，写《书名》"

→ TeamCreate("{书名}-剧组")
→ 统筹分析场景需求 → 调用角色导演选角
→ 启动因果律 Agent（主持创世会议）
→ 启动世界模型 Agent（建立world_bible）
→ 启动角色导演 + 人物创作者（选角）
→ 启动总导演（生成Spec）
→ 启动现场监理（入场许可关卡）
→ 启动角色Agent（根据场景需求）
→ 启动审查官 + 摄影 + 剪辑 + 特效

--- 进入自动循环 ---

[因果律]    与用户开卷前会议 → 生成蓝图 → 通知总导演
[总导演]    生成Scene Spec → 提交现场监理
[现场监理]  四项检查 → 签发/拒绝入场许可
[总导演]    注入锚点 → 宣布场景开始
[{角色A}]   回合制行动 → 自由即兴（锚点约束下）
[{角色B}]   回合制行动 → 自由即兴（如果在场）
[审查官]    每回合间隙检查 → OOC/AI味标记
[摄影]      持续记录
[总导演]    场景结束 → 更新角色state
[剪辑]      编排章节
[特效]      文学润色
[审查官]    全面审查 → P0/P1/P2
[5角色评审] 量化评分 → 通过/修订

--- 用户随时介入 ---

你："制造冲突" → 因果律触发天意干预
你："卡" → 总导演暂停所有Agent
你："审阅" → 查看评审报告
你："修订" → 特效启动修订循环
```
